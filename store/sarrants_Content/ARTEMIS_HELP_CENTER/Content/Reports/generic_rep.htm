Missing Acknowledgement Signals Report

Occasionally, TNQ doesn't receive eTPThe electronic Transfer Process , an automated method of transferring and tracking files. signals and we've also had reports of tasks not opening after the previous tasks closes. To help us identify and manage these situations, DJS-InfoTech created two reports. The first report, the Missing Acknowledgement Signals Report, is the one Production will run. Technical Support runs the second report each day and sends the results to the ARTEMIS users email list.

To create the Missing Acknowledgement Signals Report

NOTE:Run this report as often as your team lead indicates.

  1. Click Missing Acknowledgement Signals Report.
  2. Under Report Parameters, select the desired journal titles from the Title(s) drop-down. You may select as many as you wish.
  3. Type the Report Start Date and Report End Date or select the dates from the Calendar control.
  4. Click Submit Report. ARTEMIS generates the report and displays the results.

You can also export the result to Microsoft Excel. After ARTEMIS generates the report, click the XLS button and save the file.

Interpreting the Missing Acknowledgement Signals report

For your purposes, the important columns are Customer, Title, Identifier, TaskA Task is a function performed by a user. Tasks can contain one or more events. ARTEMIS assigns tasks to a User or Group, but only a User can close a task. Name and Production Stage.

Use this report to make sure the task following a missing eTP acknowledgement signal has opened. If you find a article, issue, ad, or FMBM(Front Matter/ Back Matter) Front and back matter, such as table of contents, index, etc. Also known as standing matter. item with no open tasks, please contact Susan ClarkSusan's internal extension is 329 and her phone number is 802-882-1618. as soon as possible.

You should ignore an report entry if any of the following are true:

  • The active workflow is past the task listed on the report.
  • The task with matching revision number has an acknowlegement signal. This means that we must have sent another signal to TNQ and received an acknowledgement on this additional signal. It can also mean that the original acknowledgement hadn't yet been processed when the report ran. In either case, you can ignore the report entry.

What About the Second Report?

V or Darcy run the second report each morning to help identify if the correct task is not open for a component. Susan Clark sends an email each morning listing which articles were found in this report. If you see an item from your journal, please alert Susan if there is an active, open task for the article. If there isn't an open task, the WorkflowThe sequence of process through which an ad, article, issue, or front and back mater pass from creation to completion. Support team can open the task you believe should be open. If you are not sure which task should be open, contact Susan Clark to review the workflow together to determine the correct open task.

Closing Memo Report

This report generates the Closing Memo for an issue.

NOTE: When you run this report, ARTEMIS displays only those remarks identified as Closing Memo in the Remarks column. Any other remarks are not listed.

To create a Closing Memo report (Generic Reports)
  1. Click Closing Memo.
  2. Select the Job Number and click Submit Report to create a Closing Memo report.
  3. Click XLS to export your results to a spreadsheet. Note that you must submit a report to generate data before you can export the date to Microsoft Excel.
  4. Click Return to Dashboard to return to the Reports page.

NOTE: Migrated data does not bring over issue sequencing data in issue sequencing order.

Work In Progress (WIP) Report

Use this report to see the work in progress.

NOTE:If you include Closed as a Status(es), this report is designed to show all articles with created date and complete date for each task.
If you don’t include Closed as a Status(es), this report is designed to show all articles and where they have only open tasks in the workflow. The tasks are listed by columns from the first one open to the last one open for the articles.

To create a Work In Progress Report (Generic Reports and Scientific Societies Reporting Dashboard)
  1. Click Work In Progress.
  2. Working from the top of the parameters list, select the Customer, Title(s), Component Type(s), Issue(s), Task(s), and Status(es).
    Issues numbered 00000 instead of a volume-issue number combination are Unassigned.
  3. Click Submit Report to create the Work In Progress report.

  4. Click XLS to export your results to a spreadsheet. Note that you must submit a report to generate data before you can export the date to Microsoft Excel.

  5. Click Return to Dashboard to return to the Reports page.
