Web Service Calls

ARTEMIS communicates with different systems to perform its tasks. To help with troubleshooting and tracing tasks, you can view Web Service Calls, or WS Calls.

To view WS Calls
  1. Search for an article.
  2. Click WS Calls under Action(s). ARTEMIS Dashboard displays the Article Web Service Calls form.

You can view this information or you can click View Call or Get Package.

 

The Article Web Service Calls form lists information about the article, its Journal, Article ID, Issue ID, Article Type, Article Title, and ComponentId.

Underneath, ARTEMIS Dashboard displays the Call Date & Time, WS Type, Workflow Task, the Call XML code, Status, and available Action(s).

You can sort on all columns except Action(s).

  • Call Date & Time—The date and time the web service call occurred.
  • WS Type—The type of Web Service call, which can be
    UnknownTAC
    FreeStyleeBill
    ProofExpressPageExpress
    JPSMathMI
  • Direction—If From Artemis, the call is out to an external system. If To Artemis, the call is into ARTEMIS from an external system.
  • Workflow Task—The specific task where this call occurred.
  • Call XML—The XML sent with the call. The XML may have been encoded within the URL or included as a POST parameter to the web service call, depending on the type of call made. ARTEMIS Dashboard displays the XML in decoded format for readability.
  • Status—The status of the call. This can be:
    AcceptedArticleNotFoundInternalServerError
    InvalidRequestXMLMissingAttribute_Request_IDMissingElement_PageCount
    OKSuccessTaskNotFound
    TitleNotFoundUnexpectedExceptionUrlDownloadFailed
    This is not a complete list. An external system may return additional status codes.
  • Actions—Links to the View Call or Get Package tasks, if available.
