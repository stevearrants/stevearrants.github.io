Files are present in this folder as long as it is in drag and drop menu

THUMBNAILS

NOT IN USAGE NOW

FIGURES

 

Contains figure which are uploaded by user (Has same name as provided by user)

When user uploads figures in submission screen

MANUSCRIPT

 

Contains manuscript uploaded by user(Has same name as provided by user)

When user uploads manuscript in submission screen

SI

 

Contains SI which are uploaded by user (Has same name as provided by user)

When user uploads Si files in submission screen

TABLES

 

Contains tables which are uploaded by user(Has same name as provided by user)

When user uploads tables in submission screen

HTML_INPUT

 

Manuscript and table inputs will be renamed as manuscript.doc and tbl[n].doc and stored in this folder

When the user clicks continue button in submission screen

SUBMISSION PACKAGE

 

NOT IN USAGE NOW

After providing First name, Last name and Email id in web interface, by default TEMP,FIGURES and SI folder will be created under HTML folder

HTML

TEMP

 

NOT IN USAGE NOW

FIGURES

figure[n].png

Uploaded figures are converted to png file format , [n] – refers to the order. These png files are used for displaying in figure sizing screen

When the user clicks continue button in submission screen

THUMBS

Thumbnails of figure[n].png will be stored in this folder. These are used for displaying in figure sizing screen

When the user clicks continue button in submission screen

SI

IMAGES

NOT IN USAGE NOW

PDF

NOT IN USAGE NOW

INPUT

Contains all SI input files (Will be renamed to Si[n] and has the file extension  as provided by user)

When the user clicks continue button in submission screen

STAMPED

(i)Si[n].pdf – individual SI files
(ii)Si.pdf – all Si files merged into single pdf
(iii)Si.stamp – Si pdf stamped

This conversion happens on loading the Meta information screen

tbl[n] files

 

Contains math type equations and symbols which are stored as png images

When the user clicks continue button in submission screen, then word2any tool will be initiated and creates table folder if there is presence of math equations and symbols

figure_caption.xml

 

Here figure captions are stored

Whenever an manuscript is uploaded, word2any converts the doc/docx files to html format. Once the html files are generated, structuring tool will fetch the figure caption from the manuscript html and place it in xml file

figure_caption_valid.xml

 

Figure captions are validated here

This file is validated during the font mapping (conversion of non Unicode to Unicode characters)

tbl[n].htm

 

Contains html file of uploaded doc table

When the user clicks continue button in submission screen, then word2any tool will be initiated and creates table html

tbl[n]_format.htm

 

Contains structured html file of tbl[n].htm

As soon as word2any creates the html file, then tagsoup creates the structured table html

manuscript.htm

 

Contains entire manuscript as submitted by author

When the user clicks continue button in submission screen, then word2any tool will be initiated and creates manuscript html

manuscript_valid.htm

 

Structured manuscript

As soon as word2any creates the html file, then tagsoup creates the structured manuscript html

Once the article is submitted,then table folder is created, If there are n number of tables submitted, then corresponding number of tbl[n] folder would be created. Below files are created during paginatepdf

Tables/ tbl[n]

tbl[n].htm

Files received from html folder

 

tbl[n]_valid.htm

Structured table

As soon as word2any creates the html file, then tagsoup creates the structured table html

tbl[n]_original.htm

This html is obtained after font mapping is done

 

tbl[n]_original_valid.htm

Structured table

As soon as word2any creates the html file, then tagsoup creates the structured table html

tbl[n].tex

These files are created during latex process in order to paginate
Tex – This file is created after typesetting of manuscript
Pdf -  Tex is converted and stored as pdf
Log – For error finding and cross checking
Aux – Contains supporting information of labels

 

tbl[n].pdf

 

tbl[n].log

 

tbl[n].aux

 

tbl[n]_crop.pdf

If the table file size is too large then it is cropped and split

 

tbl[n]_split_1.pdf

 

doc_data.txt

Has log details of table

 

start.htm

 

To initiate the process

 

front.htm

 

Contains data from metadata screen

 

body.htm

 

Has manuscript where the front matter is removed i.e., after \body

 

body_as.htm

 

Structured manuscript

 

floats.htm

 

Contains captions of figures

 

tables.htm

 

Contains captions of tables

 

end.htm

 

To end the process

 

Generated id.htm

 

All the above .htm files is combined and stored in the name of generated id.htm (Eg – epage_PDF_coc4feea27a43d3d.htm)

 

Generated id_valid.htm

 

Structured complete manuscript is present. This is obtained by combining all the 6 htm files which are present above

 

Generated id.tex

 

Typesetting is performed and stored in the format of tex

 

Generated id.pdf

 

Then finally converted to pdf

 

Generated id.log

 

Simultaneously log file is generated which could be used for cross checking and errors

 

Generated id.aux

 

Simultaneously aux file is generated which contains supporting information (labels)

 

Generated id_stamped.pdf

 

Here stamp is appended with the pdf

 

last.png

 

To calculate accurate page count, last page of the pdf is trimmed and stored as png to calculate it's height

 

Final pdf is placed in output folder

OUTPUT

Generated id.pdf

 

Si.stamp and Generated id_stamped.pdf is merged into single pdf
