Creating and Managing User Accounts

For LHC Secure only.

Most options for user accounts are available only to the System Administrator. Non-administrators are limited to changing their own password.

User Lockout

A user’s account may become locked due to one of three events:

  • Automatic lock if the user exceeded the number of successive failed login attempts.
  • Automatic lock if the user’s password expired.
  • Intentional lock by a system administrator.

To reinstate a user account, the administrator has two choices:

  • Change the status from Locked to Active: this resets the password expiration period allowing the user to login using his/her current password.
  • Enter a new password for the user (which unlocks the account) and tell the user to login with the password you have assigned him/her. Advise the user to change the password after logging in.
User Groups

Users are assigned to one of three user groups, administrator, power user, or standard user. Only Administrators have full access rights. See User Groups.
