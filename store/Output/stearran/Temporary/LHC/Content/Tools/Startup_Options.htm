Startup Options

Tools> Preferences> Startup Options 

For the best experience using the LHC, select a startup option:

  • Blank Protocol: Starts the LHC with a clean slate, that is, no instrument is selected. With this option the first task at startup is defining the instrument. This method may be preferred when using the LHC to run multiple and varied liquid-handling instruments.
  • Most Recent Protocol: Opens the last-used protocol at startup.
  • New Protocol: Starts the LHC with a new protocol (which can be based on the New Protocol template). This option is most useful when you are using the LHC to control only one instrument. Instead of selecting the instrument, defining the model and COM Port every time the LHC is launched, you can start every new protocol with this information predefined.

Using the New Protocol template, as described in Define your Startup Preferences, can be a significant timesaver.
