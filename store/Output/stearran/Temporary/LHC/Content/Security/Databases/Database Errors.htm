Database Errors

Certain conditions can cause database errors:

  • The database file is not available. Potential causes: the database is on a network and the network server is down; the computer's network cable is unplugged or WiFi control is disabled; or the file is locked by another user.
  • The database file is corrupt. Potential cause: an incomplete write operation occurred because the LHC was unexpectedly closed due to a power outage or hardware problem.
  • Verification of the database capacity failed. Potential cause: the maximum size of the database (2 Gb) has been exceeded.

Fixing the errors

First, make sure non-system issues are resolved:

  • The network is up and running and network cables are plugged in;
  • An LHC administrator is not currently performing database maintenance;
  • LHC is trying to connect to the correct database (System>Database Configuration).

Restore or replace the database file if problems persist:

  • A corrupted file or one that has exceeded its maximum capacity must be replaced with a backup or original copy. Locate the most recent backup copy of the file and Copy the Database File to the desired location.
  • If the error was database capacity, immediately clean up the file by deleting records and optimizing the database.
