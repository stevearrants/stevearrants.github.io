Database Maintenance

LHC shows the current file size.

Minimal oversight is required to keep the LHC database performing as expected. Regular backups, as with all important data, are recommended. Occasional file clean up, once a year for most organizations, is the only other administrative task required.

  • Backups: in consultation with your IT administrator, develop a plan to backup the database regularly. If you store the database on your organization's network (e.g. LAN), backup is likely already managed by your IT group. If the database resides on a stand-alone host computer, make a plan to copy the file regularly to an external storage device, e.g., external hard drive.
  • File Clean Up: in the Select System>Audit Trail>System Audit TrailAudit Trail Viewer, use the "Archive to File" and "Permanently remove records after archiving" controls to clean up, or more accurately, reduce the database file size. Keep an eye on the database file size, maximum 2 gigs, to determine when this is necessary. LHC provides a visual gauge of the file size at System>Database Configuration. When necessary, clean up the database.

For additional guidance, contact Technical Support.   

If it is not covered in your backup procedure for the database, also make a plan to backup your protocol files for safe keeping.
